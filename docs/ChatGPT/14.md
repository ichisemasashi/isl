## 14.1 計測
```lisp
(defun elapsed-seconds (thunk)
  (let ((u (internal-time-units-per-second))
        (s (get-internal-real-time)))
    (funcall thunk)
    (/ (- (get-internal-real-time) s) u)))
```

## 14.2 改善順序
1. まず計測
2. ボトルネックを特定
3. データ構造/アルゴリズムを変更
4. 再計測

## 14.3 保守性チェック
- 例外処理が抜けていないか
- 副作用点が明示されているか
- 拡張依存を隔離したか
