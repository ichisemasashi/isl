# C1.5 ★ ネイティブバイナリにする（isl-calc 5 周目）

> スパイラル **5 周目** / コンパイラ編 第 5 章 — isl-calc を高速化する

## 学習目標
- isl-calc の評価器を AOT でネイティブバイナリにビルドできる。
- JIT で式を即時コンパイル実行できる。
- インタプリタ版とコンパイル版の性能差を体感する。

## 前提知識
- strict 版評価器（[S1.4](../01-standard/04-calc-strict-edition.md)）、コンパイラ編 [C1.1](01-islc-overview.md)〜[C1.4](04-jit-run.md)。

## 内容（アウトライン）
- isl-calc を 1 ファイルにまとめ、`islc-aot` でビルド。
- 標準入力から式を読み、評価結果を出力する CLI として動かす。
- 大量の式評価でインタプリタ実行と速度比較。
- コンパイル可能な範囲（拡張機能依存があると制約が出る点）を確認。

## サンプル（スケルトン）
```sh
./bin/islc-aot -o isl-calc calc-main.lsp
echo "(+ 1 (* 2 3))" | ./isl-calc    # => 7
```

## 演習
- AOT 版と `--jit` 版で同じ式集合を評価し、起動時間と実行時間を比べる。
