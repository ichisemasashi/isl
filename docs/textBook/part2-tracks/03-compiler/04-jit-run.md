# C1.4 JIT 実行

> スパイラル **5 周目** / コンパイラ編 第 4 章

前章の AOT は「実行前にバイナリを作っておく」方式でした。もう一つ、**JIT（Just-In-Time）コンパイル**という方式があります。JIT は、プログラムを**実行する直前**に機械語へコンパイルし、そのまま走らせます。バイナリをファイルに残さない点は `--run` に似ていますが、内部は ORC v2 という LLVM の JIT エンジン（マイルストーン M11）で動いています。

## 学習目標

この章を終えると、次のことができるようになります。

- JIT モードでプログラムを即時コンパイル実行できる。
- AOT と JIT の使い分けを、それぞれの得意を踏まえて説明できる。

## 前提知識

- [C1.3 AOT ビルド](03-aot-build.md)

---

## 1. JIT で実行する

`--jit` を付けるだけです。

```sh
./bin/islc --jit examples/hello.lsp
# => 3628800
```

やっていることは「メモリ上で機械語にコンパイル → そのまま呼び出す」。ファイルとしてのバイナリは残りません。AOT が「レシピを完成品の缶詰にする」なら、JIT は「注文が入った瞬間に機械語まで作って出す」イメージです。

対話的に使える REPL もあります。

```sh
./bin/islc-jit --repl
```

こちらは、入力した式をその場で JIT コンパイルして実行する対話環境です。インタプリタの REPL（`./bin/isl`）と使い勝手は似ていますが、裏で機械語を生成している点が違います。

---

## 2. AOT と JIT の使い分け

同じ「コンパイルして実行」でも、**コンパイルの時間を誰がいつ払うか**が違います。

| | AOT（`islc-aot`） | JIT（`--jit`） |
|---|---|---|
| コンパイルのタイミング | 事前（ビルド時に一度） | 実行の直前（毎回） |
| 生成物 | ネイティブバイナリが残る | 残らない（メモリ上） |
| 起動 | 速い（もう機械語） | コンパイル分の待ちが乗る |
| 向く用途 | **配布**・繰り返し実行 | **即時実行**・対話・動的な生成 |

ポイントは、**JIT は実行のたびにコンパイル時間を払う**ことです。だから「一度ビルドして何度も配って動かす」なら AOT が有利で、「その場で一回動かす」「対話的に試す」なら JIT が手軽です。

> **小さなプログラムでの注意**: `hello.lsp` のような小さな題材だと、JIT は「コンパイルの手間」の方が「実行」より大きくなり、AOT バイナリより遅く見えることがあります。これは JIT が劣るのではなく、**コンパイル費用を実行時に前払いしている**ためです。次章のベンチマークで、この費用の乗り方を実際の数字で見ます。

---

## ★ 処理系の中では（コラム）

JIT の実装は `src/isl/compiler/jit.scm` にあり、LLVM の **ORC v2**（On-Request Compilation）API を使います。ORC は「シンボルが必要になったときにコンパイルする」遅延コンパイルの枠組みで、AOT と同じ codegen（`codegen.scm`）が生成した LLVM IR を、ファイルに落とさずメモリ上で機械語にしてリンク・実行します。つまり AOT と JIT は**同じコンパイル成果物を、いつ・どこで機械語にするかだけが違う**兄弟です。共通のパイプラインの最終段の違い、と捉えると全体像がすっきりします。

## 演習

1. **JIT 実行**: `./bin/islc --jit examples/hello.lsp` を走らせ、`3628800` を確かめよ。`islc-aot` 版と結果が同じことも確認せよ。
2. **REPL 体験**: `./bin/islc-jit --repl` を起動し、`(+ 1 (* 2 3))` を評価せよ。インタプリタ REPL（`./bin/isl`）との体感差を書き留めよ。
3. **使い分けの説明**: 「毎日 1 万回起動されるバッチ処理」と「開発中に何度も式を試す作業」で、AOT と JIT のどちらが向くか、理由とともに答えよ。

---

## まとめ

- **JIT** は実行直前にメモリ上で機械語へコンパイルして走らせる方式（ORC v2 / M11）。`--jit` と `islc-jit --repl`。
- AOT と JIT は同じパイプラインの兄弟で、**コンパイルをいつ払うか**が違う。事前に一度なら AOT、その場で毎回なら JIT。
- AOT は配布・繰り返しに、JIT は即時実行・対話に向く。小さな題材では JIT のコンパイル費用が目立つ。
