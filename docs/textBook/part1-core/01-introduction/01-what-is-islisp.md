# 1.1 ISLISP とは

> スパイラル 0 周目 / 入門編 第 1 章

プログラミング言語にはたくさんの仲間がいますが、その中でも **Lisp** は飛び抜けて変わった——そして飛び抜けて面白い——一族です。本書はその Lisp 一族の一つ **ISLISP** を、処理系 **isl** を使って学んでいきます。最初の章では、ISLISP がどんな言語なのか、そして本書でこれから何を作るのかを、ひとことずつ確かめます。

## 学習目標

この章を終えると、次のことが言えるようになります。

- ISLISP がどんな言語かを一言で説明できる。
- 本書で使う処理系 isl の位置づけ（インタプリタ＋コンパイラ）を理解する。

## 1. ISLISP は「標準化された Lisp」

ISLISP は **ISO/IEC 13816** という国際規格で定められた Lisp 系言語です。Lisp には Common Lisp や Scheme などいくつもの方言がありますが、ISLISP はその中でも「小さく・はっきりと・標準として」設計された一派です。仕様が国際規格として固まっているので、「どの処理系でも同じように動く核」がはっきりしているのが特徴です。

本書では細かい規格の条文を覚える必要はありません。「ISLISP は標準化された Lisp の一つ」——いまはそれだけ頭に入れておけば十分です。

## 2. Lisp の核心：「プログラム＝データ」

Lisp 一族に共通する最大の特徴は、**プログラムそのものがデータ（リスト）として書かれている**ことです。たとえば「1 と 2 を足す」を ISLISP ではこう書きます。

```lisp
(+ 1 2)
```

これは「`+` という記号と `1` と `2` を、丸かっこで囲んで並べたリスト」です。つまり**計算式が、そのままリストというデータ構造**になっています。この「プログラム＝データ」という考え方（S 式と呼びます）が、Lisp を特別な言語にしています。本書はこの一点を、題材を作りながら何度も体感していきます。S 式そのものは [1.4 S 式を読む](04-s-expression.md) でくわしく見ます。

## 3. 本書の処理系「isl」

本書で動かすのは **isl** という処理系です。isl は **Gauche**（Scheme 処理系）の上に実装されており、二つの顔を持ちます。

- **インタプリタ** … 式をその場で読んで評価する（REPL や `./bin/isl file`）。
- **コンパイラ** … 式を LLVM 経由でネイティブコードに変換する（`islc`、本書 5 周目で扱う）。

また isl には二つの**プロファイル**があります。

- **strict** … 規格にきっちり準拠したモード。
- **extended** … strict に独自の拡張を足したモード（本書の既定）。

プロファイルの違いは標準編（3 周目）以降で深掘りします。入門編ではすべて既定の extended のまま進めます。

## 4. 本書のゴール：「isl-calc」を 7 周作る

本書には一本の物語があります。`(+ 1 (* 2 3))` のような数式を評価する小さなインタプリタ **isl-calc** を、技術を変えながら **7 周**にわたって作り直していく、というものです。

```lisp
> (+ 1 (* 2 3))
7
```

いまはこの `7` を isl 本体が計算してくれていますが、本書を読み進めるうちに、あなたはこの「式を評価するしくみ」を**自分のコードで一から作れる**ようになります。同じ題材を繰り返し作り直すことで、リスト・再帰・オブジェクトシステム・コンパイラといった道具が、一つの物語の中で自然に身についていきます。

## ★ 処理系の中では（コラム）

上の `7` を実際に計算しているのは、リポジトリの `src/isl/core.scm` にある評価ループです。約 4700 行のこのファイルが ISLISP の心臓部です。中身がどうなっているかは、最終周（6 周目・内部編）で覗きに行きます。いまは「裏で誰かが計算してくれている」とだけ感じておけば十分です。
