# 1.3 REPL を起動する

> スパイラル 0 周目 / 入門編 第 3 章

前章で isl が起動することは確かめました。この章では、これから本書を通じて最も長く付き合う相棒——**REPL**——の使い方を身につけます。式を打ち、結果を見て、また式を打つ。この小さなリズムが Lisp の学び方の中心です。

## 学習目標

- REPL を起動し、式を評価して終了できる。
- ファイルに書いたプログラムを `./bin/isl <file>` で実行できる。

## 1. REPL とは

REPL は **Read-Eval-Print Loop**（読む・評価する・印字する・くり返す）の頭文字です。やっていることは、その名のとおり 4 ステップのくり返しです。

1. **Read** … あなたが打ち込んだ式を読み取る。
2. **Eval** … その式を評価して値を求める。
3. **Print** … 求まった値を画面に印字する。
4. **Loop** … 1 に戻り、次の入力を待つ。

コンパイルや実行ファイルの生成を挟まず、**式を打てばすぐ答えが返る**。この即時性が、言語を手で触って学ぶのに向いています。

## 2. 起動して評価する

REPL を起動します。

```sh
$ ./bin/isl
ISLISP interpreter (Gauche 0.9.15)
Profile: extended
ISLISP>
```

`ISLISP>` がプロンプトです。式を打って Enter を押すと、評価結果がすぐ下に出ます。

```lisp
ISLISP> (+ 1 2)
3
ISLISP> (* 6 7)
42
```

> **本書のお約束**: 実際のプロンプトは `ISLISP>` ですが、本書では誌面を見やすくするため、これ以降プロンプトを短く `>` と書きます。あなたの画面では `ISLISP>` と読み替えてください。

## 3. 終了する

REPL を抜けるには **Ctrl-D**（EOF, 入力の終わり）を押します。

```lisp
> (* 6 7)
42
>            ← ここで Ctrl-D
$
```

シェルのプロンプト（`$`）に戻れば終了できています。

## 4. ファイルを実行する

REPL は一行ずつ試すのに便利ですが、まとまったプログラムはファイルに書いて一気に実行します。ファイル名を引数に渡すだけです。

```sh
$ ./bin/isl examples/hello.lsp
3628800
```

このリポジトリに同梱の `examples/hello.lsp` は階乗を計算する小さなプログラムで、`3628800`（= 10 の階乗）を表示します。**REPL でもファイル実行でも、評価しているのは同じ評価器**です。一行ずつ確かめたいときは REPL、完成したものを動かすときはファイル——と使い分けます。

## ★ 処理系の中では（コラム）

REPL の正体は、`src/isl/main.scm` の中にある「Read → Eval → Print をくり返す」ループです。やっていることは本章で説明した 4 ステップそのもの。本書の 1 周目以降では、これと同じ構造を持つ「あなた自身の評価器」を少しずつ作っていきます。つまり REPL は、ゴールの完成形を先に見せてくれているお手本でもあります。
