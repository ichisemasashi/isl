# 1.3 S 式を読む

> スパイラル 0 周目 / 入門編 第 3 章

## 学習目標
- アトムとリストを区別できる。
- 前置記法（`(演算子 引数...)`）で式が表されることを理解する。
- 「プログラム＝データ」という考え方の入口に立つ。

## 前提知識
- REPL を起動できること（[1.2](02-start-repl.md)）。

## 内容（アウトライン）
- アトム（数・シンボル・文字列）とリスト `( ... )`。
- 評価ルール: リストの先頭が演算子、残りが引数。
- ネスト: `(+ 1 (* 2 3))`。
- 入れ子の式が「木（ツリー）」であること（後の AST 設計の伏線）。
- クオート `'(...)` でデータとして扱う最初の一歩。

## サンプル
```lisp
> (+ 1 (* 2 3))
7
> '(+ 1 2)        ; 評価せずデータとして見る
(+ 1 2)
```

## 演習
- `(- 10 (/ 8 2))` の結果を予想してから評価する。
