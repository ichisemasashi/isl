# 1.4 S 式を読む

> スパイラル 0 周目 / 入門編 第 4 章

ここまでで `(+ 1 2)` のような式を何度か打ちました。この丸かっこの並びには名前があります——**S 式**（S-expression）です。S 式は Lisp の文法のすべてであり、同時に Lisp の思想そのものでもあります。短い章ですが、本書全体の土台になる考え方を扱います。

## 学習目標

- アトムとリストを区別できる。
- 前置記法（`(演算子 引数...)`）で式が表されることを理解する。
- 「プログラム＝データ」という考え方の入口に立つ。

## 1. アトムとリスト

S 式は、たった 2 種類の部品でできています。

- **アトム**（atom）… それ以上分解しない最小の値。数 `1`、シンボル `+`、文字列 `"hello"` など。
- **リスト**（list）… 丸かっこで囲んで部品を並べたもの。`(+ 1 2)` など。

リストの中にはアトムだけでなく、別のリストも入れられます。この入れ子こそが S 式の表現力の源です。

## 2. 評価ルール：先頭が演算子

リストを評価するときのルールは、おどろくほど単純です。

> **リストの先頭が演算子（関数）、残りが引数。**

`(+ 1 2)` なら、「先頭の `+` を、引数 `1` と `2` に適用する」と読みます。この「演算子を前に置く」書き方を**前置記法**と呼びます。ふだんの算数の `1 + 2`（中置記法）とは順番が違うので、最初は戸惑うかもしれませんが、すぐ慣れます。

```lisp
> (+ 1 2)
3
> (* 6 7)
42
```

## 3. 入れ子は「木」になる

引数の位置には、また別のリストを書けます。

```lisp
> (+ 1 (* 2 3))
7
```

`(* 2 3)` がまず `6` になり、それから `(+ 1 6)` が `7` になります。この入れ子の構造を絵にすると、**木（ツリー）**の形をしています。

```
      (+)
      / \
     1  (*)
        / \
       2   3
```

葉（先端）が数、節（枝分かれ）が演算子つきのリストです。**S 式は木である**——この見方は、2 周目で数式を AST（抽象構文木）としてモデル化するときの伏線になります。いまは「入れ子の式は木の形をしている」と感じておけば十分です。

## 4. クオート：式を「データ」として見る

ここで Lisp ならではの一手を見ます。式の前にクオート `'` を付けると、**評価せずにデータとして**そのまま受け取れます。

```lisp
> (+ 1 2)          ; 評価する → 計算が走る
3
> '(+ 1 2)         ; クオートする → リストのまま
(+ 1 2)
```

同じ `(+ 1 2)` が、評価すれば `3`、クオートすれば `(+ 1 2)` というリスト（データ）になります。これが「**プログラム＝データ**」の正体です。`(+ 1 2)` は計算式であると同時に、`+`・`1`・`2` を並べた**ただのリスト**でもある。だからプログラムを、別のプログラムでデータとして組み立てたり調べたりできます。

本書の題材 isl-calc がまさにこれを利用します。`'(+ 1 2)` というリストを受け取り、それを自分のコードでたどって `3` を計算する——次章でその出発点を見ます。

## ★ 処理系の中では（コラム）

isl は入力テキストを、まず S 式（リスト構造）として**読み取り**、その木を評価器が**たどって**値を求めます。「読む」と「評価する」は別々の工程です。クオート `'(+ 1 2)` が計算されずにリストのまま返るのは、「読んだだけで評価していない」状態を見せてくれているのです。
