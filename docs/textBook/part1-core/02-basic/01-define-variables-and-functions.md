# 2.1 変数と関数を定義する

> スパイラル 1 周目 / 基本編 第 1 章

入門編では、REPL が「式を入れると値を返す」装置であることを確かめました。基本編（1 周目）のゴールは、その評価のしくみを自分で作る**四則電卓 isl-calc** です。この章では、そのために欠かせない二つの道具——**値に名前を付ける（束縛）**ことと、**自分の関数を定義する**こと——を身につけます。

## 学習目標

- `let` / `let*` で局所的に値を束縛できる。
- `defglobal` でグローバルな値を束縛できる。
- `defun` で関数を定義し、呼び出せる。

## 1. 値に名前を付ける——束縛

計算の途中結果に名前を付けておくと、式が読みやすくなります。これを**束縛**（binding）と呼びます。もっとも基本的なのが `let` です。

```lisp
> (let ((x 3))
    (* x x))
9
```

`(let ((x 3)) ...)` は「`x` を `3` に束縛したうえで本体を評価する」という意味です。`x` が使えるのは `let` の本体の中だけ——これを**局所束縛**といいます。

複数の束縛を一度に書くこともできます。ここで `let` と `let*` の違いが出ます。`let*` は**前の束縛を後の束縛で使える**（逐次束縛）のに対し、`let` は使えません（並行束縛）。

```lisp
> (let* ((a 2)
         (b (* a 3)))    ; a を使って b を決められる
    b)
6
```

「後ろの束縛で前の値を使いたい」なら `let*`、独立した束縛を並べるだけなら `let`、と覚えておけば十分です。

## 2. グローバルに名前を付ける——`defglobal`

`let` の束縛はその場限りですが、プログラム全体でずっと使いたい値には `defglobal` を使います。

```lisp
> (defglobal max-digit 9)
max-digit
```

定義が成功すると、ISLISP は束縛したシンボルの名前（`max-digit`）を返します。以降はどこからでも `max-digit` を参照できます。

## 3. 関数を定義する——`defun`

いよいよ本章の主役、関数定義です。`defun` で「名前・引数・本体」を与えると、再利用できる関数になります。

```lisp
> (defun square (x)
    (* x x))
square
> (square 5)
25
```

形は `(defun 名前 (引数...) 本体...)` です。本体の**最後に評価した式の値が戻り値**になります。`defun` も `defglobal` と同じく、定義に成功すると関数名を返します。

引数は何個でも取れます。たとえば 2 数の平均は次のように書けます。

```lisp
> (defun average (a b)
    (/ (+ a b) 2))
average
> (average 4 7)
11/2
```

`(/ 11 2)` が `5` でも `5.5` でもなく `11/2` になっているのは、ISLISP が整数の割り算を**正確な有理数**で扱うからです（詳しくは電卓の章と標準編で）。

## 4. この章の道具が電卓でどう使われるか

次章以降で作る電卓は、ここで学んだ道具でできています。

- 演算子シンボルを関数に対応づける表 `op->fn` … `defun` で定義。
- 式を評価する本体 `calc-eval` … `defun` で定義。
- 評価の途中結果 … `let` で束縛して読みやすくする。

つまりこの章は、電卓の「骨格」を組むための準備運動です。

## ★ 処理系の中では（コラム）

`let` の束縛や関数呼び出しの引数は、評価器の中で「環境（フレーム）」と呼ばれる入れ物に積まれていきます。`x` を参照すると、評価器はこのフレームを内側から外側へたどって `x` の値を探します。`defun` や環境の正体は 6 周目（内部編）で `core.scm` を読みながら確認します。

## 演習

1. 3 数の合計を返す `sum3` を `defun` で定義せよ。
2. `let*` を使い、`a=10`・`b=a+5`・`c=a*b` を束縛して `c` を返す式を書け。
