# 2.3 リストを組み立て・分解する

> スパイラル 1 周目 / 基本編 第 3 章

入門編で「プログラム＝データ（S 式）」を見ました。電卓が受け取る `(+ 1 2)` も、計算式であると同時に**ただのリスト**です。電卓を作るには、このリストを自在に**組み立て・分解**できる必要があります。この章でその道具を一式そろえます。

## 学習目標

- `list` / `cons` でリストを組み立てられる。
- `car` / `cdr` でリストを分解できる。
- `consp` / `null` でリストの種類と末尾を判定できる。
- 数式 `(+ 1 2)` を「演算子＋オペランドのリスト」として捉えられる。

## 1. 組み立てる——`list` と `cons`

要素を並べてリストを作るには `list` を使います。

```lisp
> (list '+ 1 2)
(+ 1 2)
```

先頭に要素を 1 つ足すには `cons`（コンス）を使います。`cons` は「既存のリストの頭に値をくっつける」操作です。

```lisp
> (cons '* '(2 3))
(* 2 3)
```

## 2. 分解する——`car` と `cdr`

リストを先頭とそれ以外に分けるのが `car` と `cdr` です。

- `car` … 先頭の要素。
- `cdr` … 先頭を除いた残りのリスト。

```lisp
> (car '(+ 1 2))      ; 先頭＝演算子
+
> (cdr '(+ 1 2))      ; 残り＝オペランド列
(1 2)
```

数式を「演算子＋オペランド」として読むと、**`car` が演算子、`cdr` がオペランド列**になります。電卓はこの見方で式を読み解きます。

## 3. 判定する——`consp` と `null`

リストを扱うときは「これは分解できるリストか？」「もう空か？」を確かめる必要があります。

- `consp` … コンス（空でないリスト）なら真。
- `null` … 空リスト `()` なら真。

```lisp
> (consp '(1 2))
#t
> (consp 5)           ; 数はコンスではない
()
> (null '())          ; 空リストか
#t
> (null '(1))
()
```

前章で学んだ「偽 ＝ `()`」がここで効いてきます。`consp` が偽のとき返るのも `()`、`null` が判定する空リストも `()`——同じ値が「偽」と「リストの終端」を兼ねているのです。

## 4. この章の道具が電卓でどう使われるか

電卓は式 `(+ 1 (* 2 3))` を、まさにこの章の道具で読み解きます。

- `consp` で「式（リスト）か、それとも数（アトム）か」を見分ける。
- `car` で演算子、`cdr` でオペランド列を取り出す。
- オペランドの中の部分式 `(* 2 3)` も、また同じ手順で分解する。

「部分式もまた同じ手順で」——この繰り返しを実現するのが、次章の**再帰**です。

## ★ 処理系の中では（コラム）

リストの正体は、`car` と `cdr` の組（コンスセル）が数珠つなぎになった構造で、末尾は空リスト `()` です。`'(+ 1 2)` は「`+` と『`1` と『`2` と `()`』』」という入れ子のセルでできています。`null` で末尾を判定できるのは、その終端が `()`（＝偽）だから——前章の真偽値の話と、ここでつながります。

## 演習

1. 式 `'(* 3 4)` から、演算子・第 1 オペランド・第 2 オペランドをそれぞれ取り出せ。
   ヒント: `car`、`(car (cdr ...))`、`(car (cdr (cdr ...)))`。
2. `(cons 1 (cons 2 (cons 3 '())))` を評価し、`(list 1 2 3)` と同じになることを確かめよ。
