# 2.4 再帰でリストをたどる

> スパイラル 1 周目 / 基本編 第 4 章

前章の最後で「部分式もまた同じ手順で分解する」と書きました。この「同じ手順を自分自身に適用する」考え方が**再帰**です。再帰は電卓の評価器そのものの骨格になります。基本編の最後の準備として、リストを再帰でたどる感覚を身につけます。

## 学習目標

- 基底ケースと再帰ケースに分けて関数を書ける。
- リストを再帰でたどって畳み込める（合計・長さなど）。
- 入れ子の式（木）を再帰でたどる発想を持つ。

## 1. 再帰の基本形

再帰関数は、いつも二つの部分でできています。

- **基底ケース**（base case）… それ以上小さくできない、答えが即わかる場合。再帰を止める。
- **再帰ケース**（recursive case）… 問題を一歩小さくして、自分自身を呼ぶ。

この二つが揃っていれば、関数は必ずいつか止まります。基底ケースを忘れると止まらなくなるので、**まず基底ケースから考える**のがコツです。

## 2. リストの合計を再帰で

リストの要素を全部足す `sum-list` を書いてみます。「空なら 0、そうでなければ先頭 ＋ 残りの合計」と考えます。

```lisp
> (defun sum-list (xs)
    (if (null xs)              ; 基底ケース：空なら 0
        0
        (+ (car xs)            ; 再帰ケース：先頭 ＋
           (sum-list (cdr xs)))))  ;           残りの合計
sum-list
> (sum-list '(1 2 3 4))
10
```

ポイントは前章の道具がそのまま効いていることです。**`null` で基底ケースを判定**し、**`cdr` でリストを一歩短く**して自分を呼ぶ。`(1 2 3 4)` → `(2 3 4)` → `(3 4)` → … → `()` と縮んでいき、最後に基底ケースで止まります。

## 3. 入れ子の式（木）を再帰でたどる

`sum-list` がたどったのは「平らなリスト」でした。電卓が相手にするのは、もう一歩進んだ**入れ子のリスト＝木**です。

```lisp
(+ 1 (* 2 3))
```

これを評価するには、`(+ 1 ...)` を計算する前に、内側の `(* 2 3)` を先に評価しなければなりません。「式を評価する関数が、部分式を評価するために自分自身を呼ぶ」——これがまさに再帰です。次章の `calc-eval` は、この形をそのままコードにしたものになります。

## 4. 末尾再帰について（軽く）

再帰には、自己呼び出しが「いちばん最後の計算」になっている**末尾再帰**という形があります。末尾再帰は効率よく実行できる利点がありますが、ここでは深入りしません。中級編で改めて扱います。いまは「再帰で書ける」ことが第一です。

## ★ 処理系の中では（コラム）

評価器そのものが、再帰で木をたどる関数です。`(+ 1 (* 2 3))` を評価するとき、評価器は内側の `(* 2 3)` を評価するために自分自身を呼び出します。次章で作る `calc-eval` がまさにこの形になり、`core.scm` にある本物の評価ループ（`eval-islisp*`）も同じ発想で動いています（6 周目で確認）。

## 演習

1. リストの要素数を返す `len` を再帰で書け（`sum-list` の「`+ (car xs)`」を「`+ 1`」に変えるだけ）。
2. リストの最大値を返す `max-of` を書け。ヒント: 基底ケースを「要素 1 個」にすると考えやすい。
