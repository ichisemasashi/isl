# 2.2 条件分岐と真偽値

> スパイラル 1 周目 / 基本編 第 2 章

電卓は「演算子が `+` なら足し算、`*` なら掛け算……」と、**場合によって振る舞いを変える**必要があります。この「場合分け」を支えるのが条件分岐です。この章では `if` と `cond`、そしてその土台になる ISLISP の真偽値を学びます。

## 学習目標

- ISLISP の真偽値（`t` と `nil`）の扱いを理解する。
- `if` で二者択一の分岐ができる。
- `cond` で多分岐を書き、`t` の既定節でまとめられる。
- `eq` でシンボルを比較できる。

## 1. ISLISP の真偽値

条件分岐の前に、「真」と「偽」が何かをはっきりさせます。ISLISP の約束はシンプルです。

- **偽**は `nil` ただ一つ。`nil` は空リスト `()` と同じもので、REPL では `()` と印字されます。
- **それ以外はすべて真**。真の代表が `t` で、REPL では `#t` と印字されます。

この約束のため、比較が偽のときは `()` が返ります。

```lisp
> (> 3 2)
#t
> (> 1 2)        ; 偽は () として返る
()
```

「偽 ＝ `()`」という一点だけ、最初は意外に感じるかもしれません。これは次章のリストと深くつながっています（コラム参照）。

## 2. `if`——二者択一

もっとも基本的な分岐が `if` です。形は `(if 条件 真の節 偽の節)`。条件が真なら真の節を、偽なら偽の節を評価します。

```lisp
> (if (> 3 2) 'yes 'no)
yes
> (if (> 1 2) 'yes 'no)
no
```

## 3. `cond`——多分岐と既定節

分岐が 3 つ以上に増えると、`if` を入れ子にするより `cond` が読みやすくなります。`cond` は節を上から順に試し、**最初に条件が真になった節**を評価します。

```lisp
> (cond ((eq 'a 'b) 1)
        ((eq 'a 'a) 2)
        (t 3))
2
```

最後の `(t ...)` 節に注目してください。`t` は「常に真」なので、**上のどの条件にも当てはまらなかったとき**の受け皿になります。これを**既定節**と呼びます。電卓では「未知の演算子が来たらエラー」をこの `t` 節で表現します。

## 4. `eq`——シンボルの比較

`cond` の条件でよく使うのが `eq` です。`eq` は二つの値が「同じもの」かを判定します。とくに**シンボル同士の比較**に使います。

```lisp
> (eq '+ '+)
#t
> (eq '+ '-)
()
```

電卓では `(eq op '+)` のように「演算子シンボルが `+` か？」を判定して振り分けます。数の等値・大小には `=` `<` `>` を使い分けます。

## 5. この章の道具が電卓でどう使われるか

電卓の心臓部 `op->fn` は、この章の道具だけで書けます。

```lisp
(cond ((eq op '+) ...)     ; eq でシンボル判定
      ((eq op '-) ...)
      ...
      (t (error ...)))     ; t 節で未知演算子を捕まえる
```

`cond` ＋ `eq` ＋ `t` 既定節——この組み合わせが、演算子の振り分け表になります。

## ★ 処理系の中では（コラム）

ISLISP では `nil` と空リスト `()` が**同じ値**です。「偽」と「空のリスト」を一つの値で兼ねているのです。これは欠点ではなく設計の妙で、おかげで次章の「リストが空か？」という判定（`null`）と、「偽か？」という判定が地続きになります。`if` でリストの末尾を検出する、といった書き方が自然にできるのはこのためです。

## 演習

1. `(if (= 0 0) 'equal 'differ)` の結果を予想してから評価せよ。
2. 引数 `n` が正なら `'pos`、負なら `'neg`、0 なら `'zero` を返す関数 `sign-of` を `cond` で書け。
